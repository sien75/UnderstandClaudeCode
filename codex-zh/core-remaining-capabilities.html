Codex Core 剩余代理能力全景

这一模块补齐 Codex Core 中不属于单一工具调用的“代理底座”：外部调用者启动或继续线程后，系统把环境、权限、技能与插件整理成模型上下文；对话过长时压缩历史；需要并行工作时派生和恢复子代理；高风险动作交给 guardian 复核；远程响应失败则在保持会话连续性的前提下重试。主线是数据如何从线程配置流入模型、又如何变成事件、审批或新的线程状态。

输入
线程配置、用户消息、当前环境与权限、历史响应项、模型流事件，以及待执行的高风险动作
输出
有界且一致的模型上下文、可恢复的子代理线程、经过策略与 guardian 审核的动作，以及持续回传给客户端的事件

上下文主链：Core 的公开入口组织线程服务；每轮开始时将环境和权限等变化编码成上下文项，再规范化调用与输出的配对关系。接近窗口上限时，本地或远程压缩会保留关键用户意图、最近工具结果和初始约束，并用摘要替换旧历史。

  env-select --> settings-update["把环境、权限和协作模式变化写入上下文"]
  settings-update --> normalize-calls["补齐调用输出并移除孤立项"]

  remote-compact --> remote-v2{"启用新版保留策略吗？"}
  remote-v2 -->|否| install-summary["筛选响应项并安装替换历史"]

  local-compact --> install-summary
  install-summary --> next-turn["下一轮重新注入必要初始上下文"]

代理生命周期主链：父线程提交独立任务后，控制器先检查深度、并发与驻留容量，再选择新建或从父历史分叉。子线程继承允许继承的环境与命令策略；完成、暂停、恢复和中断都通过统一注册状态回到父线程，旧 rollout 还可重建整棵未关闭的代理关系。

权限与可靠性主链：命令先由策略文件和托管权限配置求值；需要人工确认的风险动作可被路由到 guardian。Guardian 使用隔离的审查会话，可复用同一风险上下文并设置截止时间；拒绝会记入线程状态，暂时性失败才会重试。普通 Responses 流错误同样区分可重试、上下文超限和终止错误，避免盲目重放。

  • 技能与插件不是直接拼接任意文本：系统先发现、去重和预算化元数据，再根据显式提及或隐式匹配生成有界注入，避免上下文无限增长。
  • 环境快照区分线程级选择和本轮快照，子代理只继承策略允许传播的部分；权限变化会形成新的上下文项，而不是改写旧历史。
  • 压缩保留真实用户消息，过滤旧摘要与不适合继续发送的调用项；新版远程压缩还对文本 token 与图片数量分别设限。
  • Guardian 审查会话带复用键、取消和截止时间，防止审批阻塞主线程；拒绝状态会影响后续同轮动作。
  • 其余生产实现（MCP 资源、hook、网络代理、实时会话、rollout、shell 与 unified exec 等）已逐文件确认职责并纳入覆盖；测试、快照、静态提示模板和生成物不作为业务实现展开。
