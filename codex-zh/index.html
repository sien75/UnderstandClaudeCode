Codex CLI

这个仓库实现 OpenAI Codex CLI:一个在本机运行的 coding agent。核心产品代码集中在 codex-rs Rust workspace,外部入口包括交互式 TUI、非交互 codex exec、JSON-RPC app-server/API,以及所有入口共用的 core session、模型采样和工具执行链路。围绕核心有一圈支撑子系统:登录认证、配置管理、沙箱与命令隔离、MCP/插件/技能、状态与记忆、遥测、云端后端与云任务、网络代理。

用户能做什么

  • 运行 codex 打开终端 TUI,输入 prompt、查看流式回答、审批工具和权限请求 → 交互式终端会话 / TUI渲染与交互输入
  • 运行 codex exec 或 codex review,在脚本/CI 风格环境中提交一次任务并消费文本或 JSONL 输出 → 非交互执行任务
  • 通过 app-server JSON-RPC 创建线程、启动回合、订阅通知、读取历史 → 应用服务器线程与回合 API
  • 从任一入口触发 core agent 回合,让模型读取上下文、调用工具、处理权限并产出最终事件 → 模型回合与工具执行
  • 让模型进入"代码模式",在 V8 沙箱里写 JS 循环/并行调工具完成子任务 → 代码模式
  • 登录 ChatGPT 账号、用 API key/PAT/Bedrock、刷新 token、管理凭据存储 → 登录与认证
  • 用 codex login 之外的本地 OSS(Ollama / LM Studio)跑模型 → 云端后端与云任务
  • 连外部 MCP server 扩展工具,或把 codex 编码能力作为 MCP server 暴露给 IDE → MCP与外部工具接入
  • 安装插件/技能、配置 hooks 来扩展行为 → 插件与技能系统
  • 把远程云任务跑出的 diff apply 到本地仓库 → 云端后端与云任务
  • 恢复历史会话、查看记忆、回放 rollout → 状态与记忆 / 遥测与分析
  • 依赖沙箱隔离模型执行的命令与文件改动、审批越权操作 → 沙箱与命令执行隔离
  • 经网络代理统一管控出站流量、按 host 策略放行/拦截 → 网络代理

交互式终端会话

把用户在终端输入的一段 prompt 转成一次可审批、可流式展示、可续跑的 Codex 对话回合。

非交互执行任务

把命令行参数、stdin、图片和 schema 转成一次 headless app-server 任务,并把结果输出为文本或 JSONL。

应用服务器线程与回合 API

把外部 JSON-RPC 请求转成线程创建、回合启动、事件订阅和线程读取响应。

模型回合与工具执行

把统一入口提交的用户输入转成模型采样、工具调用、权限处理和最终事件流。

代码模式

给模型的"会写代码的子大脑":模型调 exec 工具把 JS 丢进 V8 沙箱,JS 可循环、判断、并行调工具并跨 cell 存中间值。

登录与认证

把浏览器 OAuth / 设备码登录、token 刷新与 401 恢复、固定优先级的凭据选择、多后端存储串成一条认证链。

配置加载与管理

把 7 类配置源(MDM/系统/云端/用户/profile/项目/CLI)按数字优先级叠加合并,施加硬约束与命令行覆盖,产出运行时 Config。

MCP与外部工具接入

codex 在 MCP 协议里双向身份:作为 client 聚合外部 MCP server 工具喂模型,作为 server 把编码能力暴露给 Claude/IDE,含插件接入与审批通道。

状态与记忆

管 4 个独立 SQLite 库的会话运行时状态、rollout JSONL 持久化、记忆两阶段提取合并、线程存储与 agent 父子图。

遥测与分析

把协议事件聚合成 Analytics 事实并上报,经 OTLP 导出 trace/metrics/logs,rollout 双路径持久化与确定性回放追踪。

沙箱与命令执行隔离

三平台(macOS seatbelt / Linux bubblewrap+landlock / Windows 降权)沙箱 + exec-server 执行端 + Starlark execpolicy 审批 + shell 提权拦截。

TUI渲染与交互输入

BottomPane 的 view 栈、聊天合成器与文本区、用户输入请求 overlay、会话恢复选择器——把键盘输入流转成渲染拼装与提交。

插件与技能系统

插件作为"能力集装箱"分流到 skills/tools/hooks 三个子系统,含 marketplace 安装、系统/用户技能加载、10 事件点钩子矩阵。

云端后端与云任务

Responses API 请求主链路、按 provider 的认证分发(Bearer/AgentIdentity/Bedrock SigV4)、云任务远程执行与 diff apply、模型提供商管理与本地 OSS。

网络代理

HTTP/SOCKS5 MITM 代理统一管控出站流量:host 策略评估(防 DNS rebinding)、per-host CA 签发终结 TLS、环境变量注入、uds 桥接。
