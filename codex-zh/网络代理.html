网络代理(network-proxy / uds / stdio-to-uds)

这是 codex 给 AI 跑命令时用的"网络沙箱"。当 codex 让子进程(编译、跑测试、装依赖、git 操作)联网时,不是让它直连外网,而是把子进程的环境变量全部改写成指向本地一个回环代理,再由这个代理按策略决定"这个请求能不能出去、能去哪、用什么方法"。整条链路是:子进程 → 回环 HTTP/SOCKS5 代理 → 策略判断 → 直连或上游代理或 MITM 解密 → 目标服务器。配套的 uds crate 提供跨平台 Unix domain socket 收发,stdio-to-uds 是个小桥子,把进程的标准输入输出接到 Unix socket 上,典型用途是让 git-ssh 经 SOCKS5 走代理。

核心组件分工一句话:NetworkProxy(proxy.rs)是总入口和装配器,负责绑端口、起 HTTP/SOCKS5 两个监听、给子进程写环境变量;NetworkProxyState(runtime.rs)是活的策略状态,持有配置、allow/deny glob 集合、MITM 状态、拦截记录器,所有"这个 host 能不能连"的判断都在它身上;http_proxy.rs / socks5.rs 是两个协议的入口处理函数,做协议级策略和路由;mitm.rs 在需要看 HTTPS 内层请求时,用自签 CA 证书终结 TLS、解密后再按方法/钩子策略决定放行;network_policy.rs 是统一策略评估+审计事件发射层;policy.rs 是底层 host 归一化和 glob 匹配;config.rs / state.rs / certs.rs / mitm_hook.rs / upstream.rs / connect_policy.rs / responses.rs / reasons.rs 分别管配置模型、约束校验、CA 证书、请求钩子、上游客户端、本地绑定策略、响应构造、原因字符串。

主数据流 A:HTTP 代理路径

Input:子进程的 HTTP 请求(absolute-form)或 CONNECT(HTTPS 隧道),经 HTTP_PROXY/HTTPS_PROXY 环境变量导向回环代理。
Output:放行后转发给目标(直连或上游代理或 Unix socket),或返回 403/503 拦截响应。
Notes:x-unix-socket 头是 macOS 专属逃生舱,严格 allowlist;CONNECT 在 Limited 模式或有 mitm_hook 时必须开 MITM 才能放行。

主数据流 B:MITM 解密内层

Input:CONNECT 或 SOCKS5(443)在需要 MITM 时,把 TLS 流量用自签 CA 终结。
Output:对内层 HTTP 请求做策略+钩子评估后转发,或返回拦截响应。
Notes:CA 在 codex_home/proxy/ 持久化,per-host 签 ECDSA P-256 证书;hooked host 不匹配任何钩子会被拦(mitm_hook_denied)防绕过。

主数据流 C:SOCKS5 路径

Input:子进程的 SOCKS5 TCP/UDP 请求,经 ALL_PROXY(socks5h://)导向。
Output:放行后直连或经 MITM,或返回 PermissionDenied 错误。
Notes:Limited 下 UDP 全拦、非 443 TCP 全拦;443 TCP 在 Limited 或有 hook 时走 MITM,否则直连。

策略状态机与热重载

NetworkProxyState 内部用 RwLock<ConfigState> 持有当前生效配置。每次对外查询(current_cfg/enabled/host_blocked/...)都会先调 reload_if_needed,让 ConfigReloader 判断配置源(如 config.toml)是否变了,变了就重建 ConfigState(重编 glob 集合、重编 mitm hooks、必要时重建 MitmState)。codex 自己改策略时走 add_allowed_domain/add_denied_domain/set_network_mode/replace_config_state,这些方法会先复制当前配置、改副本、用 validate_policy_against_constraints 校验不超出 managed constraints(防止子任务把自己提权),再写回并保留 blocked 缓冲。

状态机要点:NetworkMode 有 Limited/Full 两态,Limited→Full 允许(提权受 constraints 管),Full→Limited 受限。MITM 状态只在 network.mitm=true 且 build_config_state 时构造一次(MitmState 持有 CA + UpstreamClient),后续 reload 若 mitm 关掉则变 None。HostBlockDecision 是无状态计算结果(Allowed / Blocked{Denied, NotAllowed, NotAllowedLocal})。

通信边界

  • 子进程 ↔ 代理:回环 TCP,HTTP/1.1(absolute-form / CONNECT)或 SOCKS5(含 UDP associate)。子进程通过 HTTP_PROXY/HTTPS_PROXY/ALL_PROXY/NO_PROXY 等环境变量被强制指向代理;macOS 上额外写 GIT_SSH_COMMAND 让 git-ssh 经 SOCKS5。
  • 代理 ↔ 目标:直连 TCP+TLS,或经上游 HTTP 代理(读 HTTP_PROXY/HTTPS_PROXY 环境),或经 Unix socket(x-unix-socket 头,仅 macOS + allowlist)。
  • 代理 ↔ codex 主进程:NetworkProxyState 是共享 Arc,主进程直接调其方法改策略、取 blocked_snapshot/drain_blocked、设置 BlockedRequestObserver 回调;ConfigReloader trait 抽象"配置从哪来"。
  • 审计边界:所有策略决策经 tracing::event!(target: codex_otel.network_proxy) 发 codex.network_proxy.policy_decision 事件,带 conversation_id/app_version/decision/source/reason/protocol 等;BlockedRequest 同时进内存环形缓冲(上限 200 条)并经 observer 回调上报。

辅助流程:uds / stdio-to-uds 桥接

Input:进程的 stdin/stdout 或一个 Unix socket 路径。
Output:把 stdio 双向桥接到 Unix socket,或提供跨平台 UnixListener/UnixStream。
Notes:uds 在 Unix 上包 tokio 原生 UnixListener/UnixStream,目录强制 0700;在 Windows 上包 uds_windows + async_io。stdio-to-uds 接一个 socket 路径参数,连上后用 tokio::io::copy 双向搬运,stdin→socket 写完会 shutdown 写端(容忍 NotConnected 竞态)。典型场景:macOS 上 GIT_SSH_COMMAND 注入 nc -X 5 -x <socks_addr> %h %p,让 git-ssh 经 SOCKS5 出网。

关键决策表(host_blocked 决策顺序)

步骤条件结果
1deny glob 集命中Blocked(Denied)
2a非 allow_local_binding 且 host 是 loopback/私网 IP 字面量Blocked(NotAllowedLocal),除非显式 allowlist 精确命中
2b非 allow_local_binding 且 DNS 解析到私网/解析失败Blocked(NotAllowedLocal)
3allowlist 为空 或 未命中Blocked(NotAllowed)
4以上都不拦Allowed
5baseline 给 NotAllowed 且配置了 NetworkPolicyDecider交给 decider 最终裁决(可 override 成 Allow)

注:decider 只在 baseline 判 NotAllowed 时介入,且只有 Allow 才算 override(审计里 source=decider、policy_override=true)。其余 baseline 的 Denied/NotAllowedLocal 直接生效,decider 无法翻盘。

入口
NetworkProxy::builder().build() → NetworkProxy::run() 同时起 HTTP 和 SOCKS5(可选)两个 tokio task,返回 NetworkProxyHandle 可 wait/shutdown。
环境注入
NetworkProxy::apply_to_env 把 HTTP_PROXY/HTTPS_PROXY/WS_PROXY/ALL_PROXY/FTP_PROXY/NO_PROXY/NODE_USE_ENV_PROXY/ELECTRON_GET_USE_PROXY/GIT_SSH_COMMAND(macOS) 全写一遍,让子进程各种 HTTP 客户端都走代理;MITM 开启时还会把 CA 信任包路径写到 SSL_CERT_FILE/REQUESTS_CA_BUNDLE 等 10 个 key。
Unix socket 逃生舱
x-unix-socket 头只在 macOS + allow_unix_sockets 命中 + method 允许时放行,把请求转给本地 daemon(如 docker.sock),严格防远端滥用:开启 unix socket 时强制回环绑定。
CA 管理
certs.rs 在 codex_home/proxy/ 下持久化 ca.pem+ca.key(0600/0644,拒绝符号链接/组可读),签发 per-host ECDSA P-256 证书,并把原生根证书 + 自签 CA 合并成 ca-bundle-<sha>.pem 给子进程信任。
MITM 钩子
mitm_hook.rs 按 host 编译钩子,匹配 method/path_prefix/glob、query、header;命中后可剥除指定请求头、注入从 env/file 取出的 secret 头。HookedHostNoMatch(配了钩子但请求没匹配上任何钩子)会被拦(mitm_hook_denied),防止绕过。
