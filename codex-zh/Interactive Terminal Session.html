交互式终端会话

用户在终端运行 codex 后进入 TUI，在底部输入 prompt 并提交。这个场景把一段用户输入变成一次核心会话 turn：TUI 先负责终端事件、草稿、排队和本地展示，再通过 app-server 的 thread/turn 接口把输入交给 core session；模型、工具、权限请求和错误再沿 app-server 事件流回到当前可见线程，最终更新聊天记录、状态栏和审批弹窗。

输入
用户在交互式终端里输入的 prompt、图片/mention 附件、当前工作目录、模型/推理强度、审批策略和权限配置。
输出
一次正在运行或已完成的 Codex 对话 turn：包含用户消息、模型流式回答、工具执行状态、权限/审批请求、错误提示、token 用量和会话续跑信息。

  • 通信边界：TUI 通过 `AppServerSession` 访问嵌入式或远程 app-server；core session 的 `CodexThread` 被 app-server 持有，TUI 不直接消费 core 的 `EventMsg`。
  • 状态机边界：输入区会在“草稿、排队、等待 session configured、turn 运行中、审批弹窗、完成/错误”之间切换；正在运行的 turn 会让后续普通 prompt 进入队列或变成 steer。
  • 权限/沙箱边界：审批策略、权限 profile、workspace roots 和可能的 sandbox 覆盖在 `turn/start` 时随请求传入后端；需要人类确认的命令、文件变更、权限扩展会作为 `ServerRequest` 回到 TUI。
  • 生命周期边界：启动时建立 app-server bootstrap、主线程 channel 和 TUI 事件流；退出时先关闭 app-server，再清理终端显示并返回 token 用量与 resume hint。
