工具执行、审批、沙箱与 MCP

当模型请求运行本地命令、修改文件、调用 MCP 或扩展工具时，系统把响应项转换成统一工具调用，经注册表与生命周期钩子分派，再按权限策略选择审批和沙箱；结果最终被整理成模型可消费的输出。本模块只下钻工具域，不重复 CLI 主链。

输入
模型产生的函数/自定义工具调用、用户权限策略、工作目录与环境
输出
受控执行的命令输出、文件变更、MCP/扩展结果，或明确的拒绝与沙箱错误

注册表同时控制工具对模型的可见性、是否允许并行以及取消语义。扩展贡献的执行器会在每轮构造路由时加入，因此内建与扩展工具共用同一生命周期边界。

Shell 会先解析命令安全性和审批键，并保留被拒绝的读取规则；补丁则先解析、验证目标路径和写权限，再执行逐文件变更，失败时仍返回已经提交的增量，避免把部分成功误报成完全未修改。

  • 权限边界：审批要求、附加文件权限、网络许可与平台沙箱是分层的；提升权限会移除 Codex 管理的代理环境，避免把受控网络配置泄漏到无沙箱进程。
  • 通信边界：MCP 连接管理器负责服务器启动、工具发现、调用和资源读取；mcp-server 反向把 Codex 会话包装成 MCP 工具，并通过消息处理器转发审批。
  • 生命周期：Code Mode 单元可执行、等待、终止；扩展可贡献工具并观察开始/完成事件；统一 exec 会话则支持持续进程和后续 stdin。
