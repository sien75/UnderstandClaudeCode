桌面端或 IDE 通过 App Server 管理线程并运行代理回合

桌面端或 IDE 把线程管理和代理操作编码成 JSON-RPC 请求，经标准输入输出、WebSocket 或其他已配置传输交给 App Server。服务端完成连接初始化、能力与权限检查、线程和回合路由，再把生命周期事件和错误转换为通知返回客户端。

输入
客户端连接参数，以及 thread/start、thread/resume、turn/start、turn/steer、turn/interrupt 等 JSON-RPC 请求。
输出
对应的 JSON-RPC 响应，以及持续返回客户端的线程、回合、审批、错误和完成通知。

第一段数据流是进程启动和连接接入。命令行先确定传输、会话来源、认证和远程控制策略；运行时加载配置与状态库，然后按 stdio、Unix Socket、WebSocket 或远程控制建立连接。每条连接都有独立的初始化状态、实验能力、通知退订集合和串行化闸门。

第二段是 JSON-RPC 解码和连接握手。请求先按协议方法反序列化；initialize 是唯一允许在未初始化状态执行的请求。它验证客户端名称、保存实验能力和通知偏好，并返回平台信息。之后的请求若未初始化、重复初始化或使用未启用的实验字段，都会得到 JSON-RPC 错误。

第三段管理线程。thread/start 在后台加载并合并配置，检查 sandbox 与 permissions 不能同时给出，验证动态工具，然后把线程创建请求交给 core。thread/resume 优先复用仍在运行的线程，否则从显式历史、状态库记录或 rollout 文件重建；成功后自动挂接当前连接的事件监听器。core 的模型与工具循环是下层边界，这里只交接生效配置、历史、输入、动态工具和追踪上下文。

  start-result -->|否| thread-error
  start-result -->|是| attach-listener["保存线程状态并挂接事件监听器"]
  attach-listener --> thread-response["返回 thread/start 或 thread/resume 响应"]
  thread-route -->|thread/resume| resume-probe["检查线程是否正在关闭或仍在运行"]

  resume-ok -->|是| attach-listener

第四段运行回合。turn/start 将客户端输入、附加上下文和本回合设置转换成 core 的 UserInput 操作并立即返回进行中的 turn；turn/steer 只允许指向当前可引导回合；turn/interrupt 校验目标回合，普通中断要等 TurnAborted 事件再应答，启动期中断则提交成功后立即应答。

最后，线程监听器把 core 生命周期事件转换成 App Server 通知或延迟响应。通知进入出站信封后，定向消息只发给对应连接，广播只发给已初始化且未退订该方法的连接；实验字段会按连接能力过滤。慢 WebSocket 的队列满时会主动断开，断连后关闭 RPC 闸门、解绑线程监听并清理命令进程资源；正常停机还会等待后台任务并有界关闭线程。

graph TD
  core-event["core 线程产生生命周期事件"] --> lifecycle-map["转换为线程、回合、项目或错误通知"]
  lifecycle-map --> interrupt-complete["TurnAborted 同时完成待应答中断"]
  lifecycle-map --> envelope{"定向响应还是广播通知？"}
  envelope -->|定向| outbound-filter["按连接实验能力和退订项过滤"]
  envelope -->|广播| initialized-only["只选择已初始化且未退订的连接"]
  initialized-only --> outbound-filter
  outbound-filter --> queue-send{"写入连接队列成功？"}
  queue-send -->|是| client-output["经原传输编码为 JSON-RPC 返回客户端"]
  queue-send -->|队列满或关闭| disconnect["断开慢连接"]
  disconnect --> cleanup["关闭 RPC 闸门、解绑监听并清理连接资源"]
  client-close["客户端主动断连或进程停机"] --> cleanup
  • 通信边界：stdio、Unix Socket、WebSocket 和远程控制只负责把消息变成统一的 TransportEvent；业务路由不依赖具体传输。
  • 状态机边界：连接从“未初始化”单向进入“已初始化”，断连后 RPC 闸门关闭；回合从提交后的 InProgress 继续由 core 事件推进。
  • 取消语义：普通 turn/interrupt 的成功响应与 TurnAborted 事件绑定，避免“提交了中断”被误当成“回合已经终止”。
