Pi Agent Harness

This is pi — an open-source, self-extensible coding agent harness, shipped as a TypeScript monorepo. You run the pi command in your terminal; it turns a natural-language request into a multi-turn coding session: chatting with an LLM, calling tools that read and edit your files and run shell commands, and streaming the whole thing back to a terminal UI. Under the hood it is five packages that compose into one pipeline: a unified multi-provider LLM API, an agent runtime that runs the ask-the-model / run-the-tool loop, the coding-agent CLI that wires tools + system prompt + modes on top of that runtime, a terminal UI library that renders it all, and an orchestrator for running many agents as supervised child processes.

What users can do

  • Start an interactive coding chat in the terminal (pi) → Interactive Coding Agent CLI + Terminal UI Library
  • Run a one-shot prompt without entering the TUI (pi -p "fix this test") → Interactive Coding Agent CLI
  • Invoke a skill or slash command mid-session (e.g. /cl, custom skills) → Interactive Coding Agent CLI
  • Resume, fork, or import a previous session → Interactive Coding Agent CLI
  • Steer the agent mid-turn (queue a redirecting message while it works) → Agent Runtime Loop
  • Bring your own LLM provider — OpenAI, Anthropic, Google, Bedrock, Mistral, and more, via API key or OAuth → Multi-Provider LLM API
  • Run pi headless as a JSON-RPC server (used by the orchestrator and IDE integrations) → Interactive Coding Agent CLI
  • Run many agents at once: start a supervisor daemon, spawn/list/stop worker agents, stream their sessions → Multi-Agent Orchestrator

How the pieces fit together

Data flows top-to-bottom: the user types in the TUI; the coding agent builds a system prompt and tool set and hands the prompt to the agent runtime; the runtime asks the LLM through the LLM API; tool calls come back and execute against your filesystem and shell; results flow back into the loop until the agent is done. The orchestrator sits beside this, spawning many pi RPC workers and routing tasks to them.

Multi-Provider LLM API

Turn a caller's prompt + provider credentials into a streamed LLM response — one unified API across OpenAI, Anthropic, Google, Bedrock, and more.

Agent Runtime Loop

Turn a prompt plus a set of tools into a multi-turn LLM conversation — ask the model, run the tools it asks for, feed results back, repeat until done.

Interactive Coding Agent CLI

Turn the pi command into a working coding session — wires the system prompt, the read/bash/edit/write tools, and interactive/one-shot/RPC modes onto the agent runtime.

Terminal UI Library

Turn raw stdin bytes and a component tree into a rendered terminal screen — decodes key presses, diffs frames, and only redraws what changed.

Multi-Agent Orchestrator

Turn a single orchestrator serve command into a supervised fleet of coding-agent workers — spawn, route RPC, stream sessions, recover from crashes.
