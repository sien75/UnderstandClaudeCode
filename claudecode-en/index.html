Claude Code CLI (Leaked Source)

This is the leaked source code of Claude Code, Anthropic's terminal-native AI coding assistant (2026-03-31, unofficial release). A single-binary CLI running on Bun, with the entire terminal UI rendered via React + Ink. The core pipeline is: user input → CLI parsing → conversation engine (QueryEngine) → LLM API stream → if the model wants to call tools, execute them and feed results back in a loop → terminal UI rendering. TypeScript language, Bun package management, Zod validation, rule-based + auto classifier permissions, external capabilities via MCP, IDE integration via bridge, skills/plugins for extension, and CLAUDE.md for memory.

This report slices the codebase vertically into 13 business modules based on "what a user or external triggerer can do," with each module following the data flow to explain "what becomes what." The primary view is data flow, with annotations for state machines, permissions, and lifecycles where relevant. Coverage focuses on deep reading of the core pipeline, with long-tail components understood at the level of responsibilities; non-runtime assets (lock/generated terminal.js/build scripts/docker/CI) are excluded.

What Users and External Systems Can Do

  • Type claude in the terminal to launch and enter an interactive REPL → CLI Startup and Session Initialization, Terminal UI Layer
  • Ask questions and watch streaming responses (Claude may call tools multiple times along the way) → Conversation Engine and LLM Streaming, Terminal UI Layer
  • Have Claude read/write files, run shell commands, search code → Tool System and File/Shell Tools
  • Type slash commands like /commit /review /compact → Slash Command System
  • (System side, automatic before each operation) Operations pass through the permission gate: allow/ask/deny → Permission System
  • Spawn child agents, launch background tasks, and coordinate multi-agent teams → Agents, Tasks, and Multi-Agent Orchestration
  • Connect to external MCP servers to use their tools/resources, or expose Claude itself as an MCP server → MCP Subsystem
  • Use VS Code/JetBrains extensions to connect to the local CLI backend → IDE Bridge
  • Install plugins for extensibility, invoke built-in skill workflows, and have Claude remember facts persistently → Skills, Plugins, and Memory System
  • Use a claude.ai/code-style web app in the browser (memory fully in the browser) → Web Client
  • Let other AI clients browse this repo's source code through MCP → MCP Exploration Server
  • (Support layer) Message processing, session storage, bash safe parsing, plugin loading, OAuth/analytics/LSP/policy limits, etc. → Core Utilities and Service Layer Notes

CLI Startup and Session Initialization

Turns "the user types claude and hits Enter in the terminal" into "an authenticated, configured, ready-to-interact REPL session."

Conversation Engine and LLM Streaming

Turns "the user's question" into "a streaming final response" — if the model wants to call tools along the way, execute them and feed results back in a loop, while also handling thinking, retry backoff, token/cost counting, and context compression.

Tool System and File/Shell Tools

Turns "the model's request to read/write files or execute commands" into "validated by Zod, passed through the permission gate, executed, and result returned to the model" — includes FileRead/Edit/Write, Bash (AST safe parsing), Glob, Grep.

Slash Command System

Turns "the /xxx slash command the user typed" into "the corresponding execution result" — three command types (prompt sent to model / local code returning text / Ink panel returning JSX) each have their own path.

Permission System

Turns "the model wants to perform an operation that might change something" into an allow/ask/deny decision — hard rules first + four-mode split (default/plan/bypassPermissions/auto), with the auto mode having a classifier and circuit breaker.

Agents, Tasks, and Multi-Agent Orchestration

Turns "Claude wants to handle subtasks in parallel" into "spawned child agents / background shell tasks / multi-agent teams" — tasks have a full lifecycle (create → run → complete/stop), agents can message each other, and foreground/background can switch in both directions.

MCP Subsystem

Turns "tools/resources exposed by external MCP servers" into "capabilities Claude can call" (seven transports, OAuth auth, reconnect on disconnect); conversely, Claude itself can also be exposed as an MCP server to other AIs.

IDE Bridge (IDE Integration)

Turns "connection requests from VS Code/JetBrains extensions" into "a bidirectional communication session with the CLI backend" — both sides connect through a server relay, OAuth + JWT dual-token auth, message routing, permission proxying, disconnect/hibernation recovery.

Skills, Plugins, and Memory System

Turns "reusable workflows / extension capabilities / facts to remember" into "loaded and executed skills, installed plugins, and memory persisted to CLAUDE.md and recalled/synchronized across the team as needed."

Terminal UI Layer (React + Ink)

Turns "the user's keystrokes and the engine's streaming events" into "a real-time conversation interface rendered in the terminal" — the messages array is the data-flow backbone, with permission popups, tool cards, diffs, and virtual scrolling all as React components.

Web Client

Turns "a message sent from the browser" into "a streaming rendered response" — Next.js front end, /api/chat passes through to the backend SSE; memory and permissions fully in browser localStorage, with a second set of API/Worker layers prepared but not yet wired.

MCP Exploration Server

Turns "read-source-code requests from external AI clients" into "location/read/search results for files in this repo" — an independently deployable MCP server, one createServer core + STDIO/HTTP/Vercel wrappers.

Core Utilities and Service Layer Notes

Fills in support files not covered by the previous modules — message processing, session storage, bash safe parsing, attachment collection, plugin loading, OAuth, analytics, LSP, policy limits, etc. — at the level of "know the responsibilities, read the key functions."
