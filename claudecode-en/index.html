Claude Code CLI (Leaked Source Code)

This is the leaked source code of Claude Code, Anthropic's terminal-native AI coding assistant (2026-03-31, unofficial release). A single-binary CLI running on Bun, with the entire terminal interface rendered via React + Ink. The core pipeline is: user input → CLI parsing → conversation engine (QueryEngine) → LLM API streaming → if the model wants to call tools, execute them and feed the results back in a loop → terminal UI rendering. Language: TypeScript, package manager: Bun, validation: Zod, permissions: rule patterns + auto classifier, external capabilities via MCP, IDE integration via bridge, extensions via skill/plugin, memory via CLAUDE.md.

This report splits the codebase vertically into 16 business modules based on "what a user/external triggerer can do." Each module follows the data flow, explaining "what becomes what." The primary view is the data flow, with auxiliary annotations for state machines/permissions/lifecycles where relevant. Coverage focuses on a deep reading of the core pipeline and agent-design-critical subsystems, with long-tail components known by responsibility only; non-runtime assets (lock/generated terminal.js/build scripts/docker/CI) are excluded. The "Sandbox / Agent Loop Mechanics / Background and Remote Agents" modules are deep-dive supplements from an agent-design perspective.

What Users and External Actors Can Do

  • Type claude in a terminal to launch and enter REPL interaction → CLI Startup and Session Initialization, Terminal UI Layer
  • Ask questions and watch streaming replies (Claude may call tools across several turns mid-stream) → Conversation Engine and LLM Streaming, Terminal UI Layer
  • Have Claude read/write files, run shell commands, or search code → Tool System and File/Shell Tools
  • Type slash commands like /commit /review /compact → Slash Command System
  • (System side, automatically on every operation) Operations pass through a permission gate: allow / ask user / deny → Permission System
  • Spawn child agents, start background tasks, or coordinate multi-agent teams → Agents, Tasks, and Multi-Agent Orchestration
  • Connect to external MCP servers to use their tools/resources, or expose Claude itself as an MCP server → MCP Subsystem
  • Use the VS Code/JetBrains extension to connect to the local CLI backend → IDE Bridge
  • Install plugins to extend capabilities, invoke built-in skill workflows, or persist facts → Skills, Plugins, and Memory System
  • Use a claude.ai/code-style web app in the browser (memory fully in the browser) → Web App
  • Let other AI clients browse this repo's source code via MCP → MCP Exploration Server
  • (Support layer) Message processing, session storage, bash safe parsing, plugin loading, OAuth/analytics/LSP/policy limits, etc. → Core Utility Functions and Service Layer Supplement
  • (Agent design · Security) OS-level sandbox isolation before every bash/file operation: FS/network restrictions, with violations feeding back into permission decisions → Sandbox Mechanism
  • (Agent design · Loop) The query loop state machine, token-budget continuation, stop/tool hooks, and conversation history persistence → Agent Loop Mechanics
  • (Agent design · Distributed) Background forked child-agent service fleet + remote CCR agent execution plane → Background Forked Child Agents and Remote Agent Execution

CLI Startup and Session Initialization

Turns "the user types claude in a terminal and hits Enter" into "an authenticated, configured, ready-to-interact REPL session."

Conversation Engine and LLM Streaming

Turns "a single user question" into "a streaming final reply"—if the model wants to call tools mid-stream, execute them and feed the results back in a loop, while also handling thinking, retry backoff, token/cost counting, and context compression.

Tool System and File/Shell Tools

Turns "a model request to read/write files or execute commands" into "Zod-validated, permission-gated, executed, and result-returned"—including FileRead/Edit/Write, Bash (AST-safe parsing), Glob, and Grep.

Slash Command System

Turns "the user's /xxx slash command" into "the corresponding execution result"—three command types (send prompt to model / run local code returning text / pop an Ink panel returning JSX) each have their own path.

Permission System

Turns "a model-requested operation that may mutate something" into an "allow / ask user / deny" decision—hard rules first + four-mode split (default/plan/bypassPermissions/auto), with a classifier and circuit breaker in auto mode.

Agents, Tasks, and Multi-Agent Orchestration

Turns "Claude wants to process subtasks in parallel" into "spawned child agents / background shell tasks / multi-agent teams"—tasks have a full lifecycle (create→run→complete/stop), agents can message each other, and foreground/background can switch both ways.

MCP Subsystem

Turns "tools/resources exposed by an external MCP server" into "capabilities Claude can invoke" (seven transports, OAuth auth, disconnect/reconnect); conversely, Claude itself can also be exposed as an MCP server to other AIs.

IDE Bridge (IDE Integration)

Turns "a connection request from the VS Code/JetBrains extension" into "a bidirectional communication session with the CLI backend"—both sides connect through a server relay, OAuth + JWT dual-token auth, message routing, permission proxying, disconnect/hibernation recovery.

Skills, Plugins, and Memory System

Turns "reusable workflows / extension capabilities / facts to remember" into "loaded-and-executed skills, installed plugins, and memory persisted to CLAUDE.md and recalled/team-synced on demand."

Terminal UI Layer (React + Ink)

Turns "user keystrokes and engine streaming events" into "a real-time rendered conversation interface in the terminal"—with the messages array as the data-flow backbone, permission dialogs, tool cards, diffs, and virtual scrolling all as React components.

Web App

Turns "a message sent in the browser" into "a streaming rendered reply"—Next.js frontend, proxied to the backend via /api/chat as SSE; memory and permissions fully in browser localStorage, with a second set of API/Worker layers "ready but not wired up."

MCP Exploration Server

Turns "read-source-code requests from an external AI client" into "file location/read/search results for this repo"—a standalone MCP server with one createServer core + STDIO/HTTP/Vercel three shell wrappers.

Core Utility Functions and Service Layer Supplement

Fills in support files not covered by the modules above—message processing, session storage, bash safe parsing, attachment collection, plugin loading, OAuth, analytics, LSP, policy limits, etc.—to the level of "know their responsibility, deep-read key functions."

Sandbox Mechanism (OS-Level Execution Isolation)

Turns "an agent-requested bash command" into "isolated execution within FS/network restrictions, with violations feeding back into permission decisions"—the @anthropic-ai/sandbox-runtime package, deciding per-command whether to enter the sandbox, is the hard safety boundary for agent autonomous action.

Agent Loop Mechanics (State Machine · Budget · Hooks · History)

Explains the loop control flow of "one agent session" in depth—transitions terminate/continue state machine, three token-budget continuation criteria, stop/tool hook execution chain, conversation history persistence and undo. The "dictionary" and "ledger" of the agent loop.

Background Forked Child Agents and Remote Agent Execution

Covers "ambient background work" and "remote agent as a service"—a background service fleet powered by runForkedAgent + shared prompt cache (autoDream/AgentSummary/MagicDocs/SessionMemory/PromptSuggestion), plus the CCR remote execution plane (SDK message protocol, permission bridging, WebSocket reconnect, upstreamproxy credential injection).
