Pi Agent Harness

这是 pi —— 一个开源、可自扩展的编码智能体框架，以 TypeScript 单体仓库形式发布。在终端中运行 pi 命令，它会把自然语言请求转化为多轮编码会话：与 LLM 对话、调用可读写文件和运行 shell 命令的工具，并将整个过程流式回显到终端 UI。底层由五个包组成一条流水线：统一的多提供商 LLM API、负责“问模型 / 跑工具”循环的智能体运行时、在运行时之上衔接工具 + 系统提示词 + 模式的编码智能体 CLI、负责渲染的终端 UI 库，以及用于以受监管子进程方式运行多个智能体的编排器。

用户能做什么

  • 在终端中启动交互式编码聊天（pi）→ 交互式编码智能体 CLI + 终端 UI 库
  • 不进入 TUI 直接运行一次性提示（pi -p "fix this test"）→ 交互式编码智能体 CLI
  • 在会话中调用技能或斜杠命令（例如 /cl、自定义 skills）→ 交互式编码智能体 CLI
  • 恢复、派生或导入之前的会话 → 交互式编码智能体 CLI
  • 在回合中中途干预（在运行时排队一条重定向消息）→ 智能体运行时循环
  • 接入自己的 LLM 提供商 —— 通过 API key 或 OAuth 支持 OpenAI、Anthropic、Google、Bedrock、Mistral 等 → 多提供商 LLM API
  • 以无头 JSON-RPC 服务器方式运行 pi（供编排器和 IDE 集成使用）→ 交互式编码智能体 CLI
  • 同时运行多个智能体：启动监管守护进程、创建/列出/停止工作智能体、流式查看它们的会话 → 多智能体编排器

各模块如何协作

数据流自上而下：用户在 TUI 中输入；编码智能体 构建系统提示词和工具集，再交给 智能体运行时；运行时通过 LLM API 调用 LLM；返回的工具调用针对文件系统和 shell 执行；执行结果再流回循环，直到智能体完成。编排器 独立于这一主流程，负责派生多个 pi RPC 工作进程并为其路由任务。

多提供商 LLM API

把调用方的提示词 + 提供商凭证转化为流式 LLM 响应 —— 一套统一 API 覆盖 OpenAI、Anthropic、Google、Bedrock 等。

智能体运行时循环

把提示词和一组工具转化为多轮 LLM 对话 —— 问模型、跑它要求的工具、把结果喂回去，重复直到完成。

交互式编码智能体 CLI

把 pi 命令变成一个可用的编码会话 —— 衔接系统提示词、read/bash/edit/write 工具，以及交互/一次性/RPC 模式。

终端 UI 库

把原始 stdin 字节和组件树渲染成终端屏幕 —— 解码按键、diff 帧、只重绘变化的部分。

多智能体编排器

把单个 orchestrator serve 命令变成受监管的智能体工作集群 —— 派生、路由 RPC、流式会话、崩溃恢复。
