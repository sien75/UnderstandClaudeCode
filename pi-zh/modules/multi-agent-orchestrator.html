多智能体编排器

@earendil-works/pi-orchestrator 包是一个监管进程，负责生成并管理多个编码智能体子进程，在调用方（CLI 或外部客户端）与这些工作进程之间路由任务和 RPC 流量，并协调它们的生命周期。从用户角度看，只需运行一次 orchestrator serve 即可启动长期运行的监管守护进程；随后所有其他命令——list、spawn、status、stop、rpc、rpc-stream——都是轻量客户端，通过 Unix 套接字连接到该守护进程，发送 JSON-Lines 请求并打印 JSON 响应。监管进程本身将每个智能体作为子进程生成，并通过其 stdin/stdout 使用 @earendil-works/pi-coding-agent 定义的同一 JSON-Lines RPC 协议与之通信。此外，监管进程还可以选择将自己和每个 Pi 工作进程注册到远程 "Radius" 在线服务，以便其他机器发现并访问该编排器。

cli.ts — 命令分发器
解析 process.argv，并路由到 serve()（启动守护进程）或调用 sendIpcRequest 的某个客户端命令。对于 rpc-stream，它会打开原始双向套接字，并将 stdin 的 JSON-Lines 管道输入其中。输入：argv 标志。输出：打印的 JSON 响应或一个运行中的守护进程。
serve.ts — 守护进程入口
启动 IPC 套接字服务器，在重启后恢复持久化的实例状态，可选择启动 Radius 在线服务，然后永久阻塞，直到信号或致命错误触发优雅关闭：关闭服务器、停止所有工作进程并删除套接字文件。输入：无（副作用：一个运行中的套接字）。输出：一个长期运行的进程。
ipc/server.ts — 套接字服务器
接受 Unix 套接字连接，读取以换行符分隔的 JSON，将每一行解析为 OrchestratorRequest，分派给处理器，并写回 JSON 响应。对于 rpc_stream 请求，它会将连接升级为长期存在的双向桥接：将智能体会话事件和 UI 请求扇出给客户端，并按串行方式排队入站 RPC 命令。输入：套接字上的 JSON-Lines 请求。输出：JSON-Lines 响应和流式事件。
ipc/client.ts — 套接字客户端
每个非 serve 的 CLI 子命令使用的一次性辅助函数：打开到编排器套接字的连接，写入一个请求，读取一个响应并返回。输入：一个 OrchestratorRequest。输出：一个 OrchestratorResponse。
ipc/protocol.ts — 有线格式
定义请求/响应的可辨识联合类型，以及 encodeMessage/parseRequestLine/parseResponseLine 辅助函数。套接字上的每个字节都是 JSON.stringify(msg) + "\n"。
handler.ts — 请求路由器
根据 request.type 进行分支，并委托给对应的 supervisor 方法，将 InstanceRecord 映射为便于传输的 InstanceSummary。同时暴露 openRpcStream，将 supervisor 的流句柄适配为面向套接字的回调形态。
supervisor.ts — 生命周期与路由核心
持有内存中的 liveInstances 映射（每个工作进程对应一个 LiveInstance，将持久化的 InstanceRecord 绑定到活动的 RpcProcessInstance、订阅者集合以及 UI 请求处理器）。负责生成、停止、RPC 路由、流订阅、崩溃恢复和关闭。将实例元数据持久化到 instances.json，并在改变会话状态的 RPC 调用后刷新。将自己配置为 Radius 协调器。
rpc-process.ts — 子进程包装器
为每个工作进程生成一个 pi --mode rpc（或 pi-coding-agent/rpc-entry）子进程，通过 stdin/stdout 成帧 JSON-Lines，按 id 将响应与待处理请求关联，向监听者扇出会话事件和 extension_ui_request 消息，并检测子进程退出/错误。输入：RpcCommand。输出：RpcResponse + 异步 AgentSessionEvent 流。外部边界：一个 Node 子进程。
storage.ts — 持久化
在编排器目录下读写 instances.json 和 machine.json。纯 JSON 文件 IO，无缓存。
config.ts — 路径与环境
解析编排器目录（PI_ORCHESTRATOR_DIR 或 ~/.pi/orchestrator）、套接字路径、instances/machine/auth 路径、包版本，以及决定子进程生成命令的 isBunBinary 标志。
radius.ts — 远程在线服务
与 Radius 服务的可选集成：注册本机和每个 Pi（工作进程），以指数退避 + 抖动方式运行心跳，并在多次 404 后重新注册。外部边界：Radius HTTP API。
types.ts — 共享类型
InstanceStatus（"starting" | "online" | "stopping" | "stopped" | "error"）、InstanceRecord、MachineRecord、RadiusRegistration。
