交互式编码智能体 CLI（`pi`）

本模块是面向用户的包 @earendil-works/pi-coding-agent —— 即 pi 命令行工具。它是将整个 monorepo 串联起来的集成层：负责 CLI 入口、三种运行时模式（交互式 TUI、单次打印、无头 JSON-RPC）、内置文件系统/Shell 工具（read、bash、edit、write，以及可选的 grep/find/ls）、会话与配置管理，以及系统提示词。它驱动 @earendil-works/pi-agent-core 运行时，后者再通过 @earendil-works/pi-ai 与 LLM 提供商通信。

用户可以使用 pi 做什么：

  • 交互式聊天：在终端运行 pi 进入全屏 TUI，流式模型输出、文件/bash/edit/write 工具调用内联渲染、斜杠命令、skills、提示模板、模型切换（Ctrl+P）以及流中 steering。
  • 单次提示：pi -p "list .ts files"（或 pi --mode json "..."）以非交互方式运行单条提示，打印最终答案（或 JSON 事件流）后退出。管道输入的 stdin 会被视为提示的一部分。
  • 无头 RPC：pi --mode rpc 将进程变为基于 stdin/stdout 的 JSON 行服务器 —— 编排器（编辑器扩展、CI 运行器、另一个智能体）可发送 prompt、steer、abort、switch_session、fork 等命令，并接收流式会话事件。
  • 会话管理：恢复（--continue / --resume）、按 id/路径打开（--session）、分叉（--fork）、命名（--name），或运行临时会话（--no-session）。
  • 斜杠命令 / skills：在提示中以 / 开头的是扩展命令；/skill:name args 内联 skill 文件；/template args 展开提示模板。
  • 包/配置子命令：pi install <source>、pi remove、pi update、pi list、pi config 用于管理扩展和设置；这些在 agent 运行时构建之前就会退出。
  • 导出：pi --export session.jsonl [output.html] 将保存的会话渲染为 HTML 后退出。

用户运行 pi ... 的瞬间，整个链路就被触发：cli.ts 把 process.argv 交给 main()，后者解析参数、构建会话 + agent 运行时，然后分派到三种模式之一。模式将用户文本喂给 AgentSession.prompt()，由它驱动 pi-agent-core 中的 Agent 循环；Agent 调用 LLM 提供商、针对文件系统/Shell 执行工具调用，并在模型停止请求工具前循环。流式事件通过 AgentSession 回传给模式的监听器，由它渲染（TUI）或序列化（print/rpc）。

模块
pi CLI 将用户键入的请求（以及文件、管道 stdin 或 JSON-RPC 命令）转换为一系列 LLM 轮次，并与文件系统/Shell 工具执行交错，最终输出渲染后的答案 —— 交互式、打印式或以 JSON 事件流形式。
输入
CLI 参数 + 提示（键入文本、@file 引用、管道 stdin 或 JSON-RPC prompt 命令），针对当前工作目录求值。
输出
交互模式：实时 TUI。打印模式：stdout 上的最终助手文本（或 JSON 事件流）。RPC 模式：stdout 上的 JSON 行事件/响应流。副作用：文件读/写/编辑、Shell 命令执行。
备注
本包本身不实现 LLM 轮次循环，该逻辑位于 @earendil-works/pi-agent-core（Agent）。本包提供工具、系统提示词、认证/设置/模型注册表、会话持久化、扩展钩子和面向用户的模式。外部边界：LLM 提供商通过配置在 Agent 上的 streamFn 中的 streamSimple 接入；文件系统和 Shell 只能通过内置工具接入。

图 3 — 提示循环（AgentSession → Agent → 工具 → LLM）

标记的状态机

  • Agent 流式状态（agent.state.isStreaming）：idle → streaming（在 agent.prompt/agent.continue 期间）→ idle。AgentSession.isStreaming 决定 prompt() 是通过 steer/followUp 排队还是立即发送。这是控制用户输入路由的核心状态机。
  • 会话生命周期（AgentSessionRuntime）：运行时创建后，可通过 newSession / switchSession / fork / importFromJsonl / reload / dispose 拆除并替换。每次替换会发出 session_before_switch，拆除（发出带原因 new/resume/fork/reload/quit 的 session_shutdown），重新运行工厂并重新绑定。
  • 自动重试状态（_retryAttempt）：遇到可重试错误（过载/限流/服务器，但不包括上下文溢出）时，发出 auto_retry_start 并指数退避，继续 agent；用尽时发出 auto_retry_end 且 success=false。
  • 压缩状态：当最后一条助手消息溢出上下文时，_checkCompaction 运行一次压缩轮次（发出 compaction_start/compaction_end）并用压缩后的上下文继续 agent。上下文溢出错误被路由到压缩，而非重试。
  • 项目信任状态：untrusted → trusted，按 cwd 缓存在 projectTrustByCwd 中；--approve/--no-approve 可覆盖；交互模式可以提示。
  • RPC 关闭状态（shutdownRequested/shuttingDown）：shutdown 命令或 stdin 结束会设置标志；当前命令完成后，checkShutdownRequested 执行完整拆除。

外部边界说明

  • LLM 提供商：仅在 sdk.ts 中为 Agent 配置的 streamFn 内部接入。它调用 modelRegistry.getApiKeyAndHeaders(model) 获取凭证，合并提供商归因标头，允许扩展通过 before_provider_headers 钩子调整标头，然后调用 @earendil-works/pi-ai 的 streamSimple。onPayload/onResponse 钩子让扩展观察/修改外发请求和传入响应状态/标头。
  • 文件系统与 Shell：仅通过七个内置工具接入。bash 工具生成真正的 Shell（使用 getShellEnv 的 spawn，以及可配置的 shellPath/commandPrefix），跟踪进程树以支持中止/超时，并流式传输 stdout/stderr。read/edit/write/grep/find/ls 操作 cwd 下的真实文件。文件修改类工具通过 withFileMutationQueue 串行化。
  • TUI 渲染：InteractiveMode 订阅 AgentSession 事件，并通过 @earendil-works/pi-tui 组件渲染流式文本、工具调用和审批。TUI 还会捕获用户输入（steering、斜杠命令、模型切换）并回传给 session.prompt。
  • 会话持久化：SessionManager 将每个 agent 事件追加到会话目录下的 JSONL 会话文件；在 resume/continue/fork 时从该文件重建消息上下文。这正是 --continue、--resume 和 /fork 能够工作的原因。
  • 扩展：由 DefaultResourceLoader 加载；它们可以注册斜杠命令、自定义工具、prompt-template/skill/theme 提供方、模型提供方，并挂接到 input、tool_call、tool_result、before_agent_start、before_provider_request、after_provider_response、session_start/session_shutdown、session_before_switch/session_before_fork 以及 context 转换事件。
