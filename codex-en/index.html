Codex CLI

This repository implements the OpenAI Codex CLI: a coding agent that runs locally. The core product code lives in the codex-rs Rust workspace. External entry points include the interactive TUI, non-interactive codex exec, the JSON-RPC app-server/API, plus the core session, model sampling, and tool execution pipeline shared by every entry point. Surrounding the core is a ring of supporting subsystems: login/authentication, configuration management, sandbox and command isolation, MCP/plugins/skills, state and memory, telemetry, cloud backend and cloud tasks, and network proxy.

What users can do

  • Run codex to open the terminal TUI, enter a prompt, view streaming responses, and approve tools and permission requests → Interactive Terminal Session / TUI Rendering and Interactive Input
  • Run codex exec or codex review to submit a one-off task in a script/CI-style environment and consume text or JSONL output → Non-Interactive Task Execution
  • Create threads, start turns, subscribe to notifications, and read history via the app-server JSON-RPC API → App-Server Threads and Turn API
  • Trigger a core agent turn from any entry point so the model reads context, calls tools, handles permissions, and produces final events → Model Turn and Tool Execution
  • Let the model enter "code mode" and write JS loops/parallel tool calls inside a V8 sandbox to finish subtasks → Code Mode
  • Log in with a ChatGPT account, use an API key/PAT/Bedrock, refresh tokens, and manage credential storage → Login and Authentication
  • Run models locally via OSS (Ollama / LM Studio) without using codex login → Cloud Backend and Cloud Tasks
  • Connect external MCP servers to extend tools, or expose Codex coding capabilities as an MCP server to an IDE → MCP and External Tool Integration
  • Install plugins/skills and configure hooks to extend behavior → Plugin and Skill System
  • Apply diffs produced by remote cloud tasks back to the local repository → Cloud Backend and Cloud Tasks
  • Resume historical sessions, view memories, and replay rollouts → State and Memory / Telemetry and Analytics
  • Rely on sandbox isolation for model-executed commands and file changes, and approve overreach operations → Sandbox and Command Execution Isolation
  • Route outbound traffic through a unified network proxy and allow/block by host policy → Network Proxy

Interactive Terminal Session

Turn a prompt the user types in the terminal into an approvable, streamable, resumable Codex conversation turn.

Non-Interactive Task Execution

Turn command-line arguments, stdin, images, and schemas into a headless app-server task and output the result as text or JSONL.

App-Server Threads and Turn API

Translate external JSON-RPC requests into thread creation, turn startup, event subscription, and thread-read responses.

Model Turn and Tool Execution

Translate user input submitted through a unified entry point into model sampling, tool calls, permission handling, and a final event stream.

Code Mode

A "code-writing sub-brain" for the model: the model calls the exec tool to drop JS into a V8 sandbox, where the JS can loop, branch, call tools in parallel, and store intermediate values across cells.

Login and Authentication

Chain browser OAuth / device-code login, token refresh and 401 recovery, fixed-priority credential selection, and multi-backend storage into a single authentication flow.

Configuration Loading and Management

Merge seven kinds of configuration sources (MDM/system/cloud/user/profile/project/CLI) by numeric precedence, apply hard constraints and command-line overrides, and produce the runtime Config.

MCP and External Tool Integration

Codex's dual role in the MCP protocol: as a client it aggregates external MCP server tools and feeds them to the model; as a server it exposes coding capabilities to Claude/IDEs, including plugin integration and an approval channel.

State and Memory

Manage four independent SQLite databases for session runtime state, rollout JSONL persistence, two-stage memory extraction and merging, thread storage, and agent parent-child graphs.

Telemetry and Analytics

Aggregate protocol events into Analytics facts and report them, export traces/metrics/logs via OTLP, and provide dual-path rollout persistence and deterministic replay tracking.

Sandbox and Command Execution Isolation

Three-platform sandboxing (macOS seatbelt / Linux bubblewrap+landlock / Windows de-elevation) plus exec-server execution, Starlark execpolicy approval, and shell escalation interception.

TUI Rendering and Interactive Input

BottomPane view stack, chat composer and text area, user-input-request overlay, and session-resume selector—turning keyboard input into rendering assembly and submission.

Plugin and Skill System

Plugins act as "capability containers" split into skills/tools/hooks subsystems, including marketplace installation, system/user skill loading, and a 10-event hook matrix.

Cloud Backend and Cloud Tasks

Main Responses API request path, provider-specific auth distribution (Bearer/AgentIdentity/Bedrock SigV4), remote cloud task execution and diff apply, and model-provider management plus local OSS.

Network Proxy

HTTP/SOCKS5 MITM proxy that unifies outbound traffic control: host policy evaluation (anti DNS rebinding), per-host CA issuance and TLS termination, environment-variable injection, and UDS bridging.
