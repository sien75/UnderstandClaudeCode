OpenAI Codex

This is the code understanding map for the Codex open-source monorepo. The project is primarily written in Rust, providing a CLI, terminal interactive interface, and App Server; TypeScript and Python SDKs embed these capabilities into other applications. The core task is to transform user goals into contextualized model requests, invoke local or remote tools under approval and sandbox constraints, and persistently save thread history, state, and events.

What Users and External Systems Can Do

  • Enter tasks from CLI, resume sessions, and run full agent turns → A user initiates and completes an agent turn from the CLI
  • Edit input in the terminal, view streaming output, and respond to approvals → Terminal interaction
  • Create threads, start/guide/interrupt turns via RPC from desktop or IDE → App Server RPC
  • Embed, stream-consume, and control Codex through TypeScript or Python SDKs → SDK Integration and Runtime
  • Choose configuration, login methods, and model providers → Configuration, Authentication, and Model Connection
  • Let the agent execute commands, apply patches, invoke MCP, under approval and sandbox protection → Tool Execution, Approval, Sandbox, and MCP
  • Save, restore, compress, and migrate thread history → Core State and History
  • Install plugins, skills, and extensions, connect cloud tasks, network proxies, and telemetry → Supporting Services
