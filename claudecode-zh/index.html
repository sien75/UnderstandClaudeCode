Claude Code CLI（泄露源码）

这是 Anthropic 的终端原生 AI 编程助手 Claude Code 的泄露源码（2026-03-31，非官方发布）。一个 Bun 运行的单二进制 CLI，整个终端界面用 React + Ink 渲染。核心管道是：用户输入 → CLI 解析 → 对话引擎(QueryEngine) → 调 LLM API 流式返回 → 若模型要调工具就执行工具把结果喂回去循环 → 终端 UI 渲染。语言 TypeScript，包管理 Bun，校验 Zod，权限用规则模式 + auto 分类器，外部能力靠 MCP，IDE 集成靠 bridge，扩展靠 skill/plugin，记忆靠 CLAUDE.md。

本报告按"用户/外部触发者能做什么"纵向切成 16 个业务模块，每个模块顺着数据流讲清"什么变成什么"。主视图为数据流，遇到状态机/权限/生命周期等模式时辅助标注。覆盖以核心管道深度精读为主、长尾知晓职责为辅，非运行时资产（lock/生成的 terminal.js/构建脚本/docker/CI）已排除。其中"沙箱 / agent 循环机制 / 后台与远程 agent"三块为 agent 设计视角的补深模块。

用户与外部能做什么

  • 在终端敲 claude 启动、进入 REPL 交互 → CLI 启动与会话初始化、终端 UI 层
  • 提问、看流式回复（中途 Claude 可能自己调工具若干轮）→ 对话引擎与 LLM 流式调用、终端 UI 层
  • 让 Claude 读写文件、跑 shell 命令、搜索代码 → 工具系统与文件/Shell 工具
  • 敲 /commit /review /compact 等斜杠命令 → 斜杠命令系统
  • （系统侧，每次操作自动发生）操作前过权限闸门：放行/问用户/拒绝 → 权限系统
  • 派子智能体、起后台任务、组多智能体团队协同 → 智能体、任务与多智能体编排
  • 连接外部 MCP server 用其工具/资源，或把 Claude 自己暴露成 MCP server → MCP 子系统
  • 用 VS Code/JetBrains 扩展连上本地 CLI 后端 → IDE Bridge
  • 装 plugin 扩展能力、调预置 skill 工作流、让 Claude 持久记住事实 → 技能、插件与记忆系统
  • 在浏览器用 claude.ai/code 风格的网页版（记忆全在浏览器）→ Web 端
  • 让别的 AI 客户端通过 MCP 浏览本仓库源码 → MCP 探索服务器
  • （支撑层）消息处理、会话存储、bash 安全解析、插件加载、OAuth/埋点/LSP/策略限制等 → 核心工具函数与服务层补充
  • （agent 设计·安全）每次 bash/文件操作前的 OS 级沙箱隔离：FS/网络限制、违规回流权限 → 沙箱机制
  • （agent 设计·循环）query 循环的状态机、token 预算续写、stop/工具钩子、对话历史落盘 → Agent 循环机制
  • （agent 设计·分布式）后台 forked 子智能体服务群 + 远程 CCR agent 执行面 → 后台 Forked 子智能体与远程 Agent 执行

CLI 启动与会话初始化

把"用户在终端敲下 claude 回车"变成"一个鉴权过、配置加载好、就绪可交互的 REPL 会话"。

对话引擎与 LLM 流式调用

把"用户的一句提问"变成"流式返回的最终回复"——中途若模型要调工具，就执行工具把结果喂回去循环，还要管 thinking、重试退避、token/成本计数、上下文压缩。

工具系统与文件/Shell 工具

把"模型要读写文件或执行命令的请求"变成"经 Zod 校验、过权限闸门、执行完、结果回传给模型"——含 FileRead/Edit/Write、Bash（AST 安全解析）、Glob、Grep。

斜杠命令系统

把"用户敲的 /xxx 斜杠命令"变成"对应的执行结果"——三类命令（送提示词给模型 / 跑本地代码返文本 / 弹 Ink 面板返 JSX）各有路径。

权限系统

把"模型要执行一个可能改东西的操作"变成"放行 / 问用户 / 拒绝"的决策——硬规则前置 + 四模式分流（default/plan/bypassPermissions/auto），auto 模式有分类器与熔断。

智能体、任务与多智能体编排

把"Claude 想并行处理子任务"变成"派生的子智能体 / 后台 shell 任务 / 多智能体团队"——任务有完整生命周期（创建→运行→完成/停止），智能体间能互发消息，前台后台可逆切换。

MCP 子系统

把"外部 MCP server 暴露的工具/资源"变成"Claude 能调用的能力"（七种 transport、OAuth 鉴权、断连重连）；反过来也能把 Claude 自己作为 MCP server 暴露给别的 AI。

IDE Bridge（IDE 集成）

把"VS Code/JetBrains 扩展的连接请求"变成"CLI 后端的一个双向通信会话"——双方都连服务端中转，OAuth + JWT 双 token 鉴权，消息路由、权限代理、断连/休眠恢复。

技能、插件与记忆系统

把"可复用工作流 / 扩展能力 / 要记住的事实"分别变成"加载执行的 skill、安装的 plugin、持久化到 CLAUDE.md 并按需回忆/团队同步的记忆"。

终端 UI 层（React + Ink）

把"用户的敲键与引擎的流式事件"变成"终端里实时渲染的对话界面"——以 messages 数组为数据流主轴，权限弹窗、工具卡片、diff、虚拟滚动皆 React 组件。

Web 端

把"浏览器里发的一条消息"变成"流式渲染的回复"——Next.js 前端，走 /api/chat 透传后端 SSE；记忆与权限全在浏览器 localStorage，存在"备好但未接线"的第二套 API/Worker 层。

MCP 探索服务器

把"外部 AI 客户端的读源码请求"变成"对本仓库文件的定位/读取/搜索结果"——独立部署的 MCP server，一个 createServer 内核 + STDIO/HTTP/Vercel 三种外壳。

核心工具函数与服务层补充

把前面模块没覆盖的消息处理、会话存储、bash 安全解析、附件收集、插件加载、OAuth、埋点、LSP、策略限制等支撑文件，补齐到"知晓职责、关键函数精读"。

沙箱机制（OS 级执行隔离）

把"agent 要跑的一条 bash 命令"变成"在 FS/网络限制内隔离执行、违规回流权限决策"——包 @anthropic-ai/sandbox-runtime，按命令决策进不进沙箱，是 agent 自主动手的硬安全边界。

Agent 循环机制（状态机·预算·钩子·历史）

把"一次 agent 会话"的循环控制流讲透——transitions 终止/继续状态机、token 预算续写三重判据、stop/工具钩子执行链、对话历史落盘与撤销。agent loop 的"字典"与"账本"。

后台 Forked 子智能体与远程 Agent 执行

把"环境式后台工作"与"远程 agent 即服务"两块讲清——runForkedAgent+共享 prompt cache 的后台服务群（autoDream/AgentSummary/MagicDocs/SessionMemory/PromptSuggestion），加 CCR 远程执行面（SDK 消息协议、权限桥接、WebSocket 重连、upstreamproxy 凭据注入）。
